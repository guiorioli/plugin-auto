# Claude Memory

## Instruções Obrigatórias
Ler os arquivos REQUIREMENTS.md, TODO.md, DONE.md e README.md para contextualizar o projeto
 - O arquivo REQUIREMENTS contém todos os requisitos do projeto, em formato de PRD
 - O arquivo TODO contém uma lista de tasks TO DO
 - O arquivo DONE recebe as tasks concluídas vindas do TODO
 - O arquivo README contém a arquitetura do projeto (tecnologias, linguagens, frameworks, servidores, estrutura de pastas) e a descrição de como executar (onboarding)
 
Sempre que um novo requisito for solicitado, atualizar o arquivo REQUIREMENTS, quebrar em tasks e adicioná-las ao TODO. Perguntar antes de iniciar o desenvolvimento.
Se uma tarefa fora do backlog for realizada, atualizar DONE e REQUIREMENTS quando aplicável
Caso uma decisão de arquitetura seja modificada, atualizar o README
Quando uma task for realizada:
 - verificar necessidade de criação de teste unitário para ela
 - executar testes, corrigir erros
 - mover as tasks concluídas de TODO para DONE
 - realizar commit e push, caso o git esteja configurado
